Project Overview

Visual overview of the marker production line concept.

This project simulates a manufacturing line that assembles ink markers. The backend controls the production sequence, the HMI gives operator control, and the system detects defective products such as cracked bodies, underfilled reservoirs, missing tips and faulty caps.

Production Line Stages

1. Marker Body Molding

The plastic marker body is created. Defect example: cracked body.

2. Ink Reservoir Filling

The internal reservoir is filled with ink. Defect example: underfilled cartridge.

3. Tip Insertion

The writing tip is inserted and aligned. Defect example: missing or misaligned tip.

4. Cap Assembly

The cap is attached to the marker. Defect example: cap not locked correctly.

5. Quality Control and Packaging

The final marker is checked and packaged. Defective markers are rejected.

System Architecture

Architecture separating physical stations, backend logic, HMI and data pipeline.

The design separates the operator interface from the production backend. This makes the project easier to maintain and reflects the structure of industrial software, where HMI, backend logic, database and dashboard are separated into clear layers.

HMI Frontend

The HMI includes Start, Stop and Reset buttons, live machine state, current station, counters for good and defective products, and an error display for defects.

Docker, InfluxDB and Grafana

InfluxDB stores time-series production values and Grafana visualizes the values as a dashboard. Docker Compose defines the monitoring services so the stack can be started consistently.

Question Bank and Concepts

What is an HMI?
An HMI is the Human-Machine Interface used by an operator to control and monitor a machine.
What is the backend?
The backend contains the logic, states, production rules, defect decisions and data handling.
What is Docker used for?
Docker packages services like InfluxDB and Grafana into containers that are easier to start and reproduce.
What is OEE?
OEE means Overall Equipment Effectiveness and is calculated using availability, performance and quality.
What is Git?
Git is a version control system used to track file changes and development history.
